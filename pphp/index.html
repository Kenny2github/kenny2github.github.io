Python PPHP: Hypertext Preprocessor

PPHP 3.0.0

Changes in this release:

+The database is now stored in a single file (~/.pphp/__DATABASE__.json or %appdata%/.pphp/__DATABASE__.json), with each server's database being stored under one database key.
 +This means that when you first run your server, you will be prompted for a database key to store this server's data under. If the root is changed, you will be prompted again - but you can easily specify the same db key to continue using the data stored there.
 +To store the db key, a hidden file (.pphp-config) will be created directly under the root directory. This file simply contains the database key.
*The "do" function now requires two arguments (but no more): html and server. "html" should be the content of the page; "server" should be the server instance.
+The actual (recommended use) server built-in now uses the "thread" module instead of "threading" (although it still uses threading for its lock object).
 +The actual server built-in can now actually stop each individual port. Just enter the port number into the console to stop the server on that port.
 -This however means that IDLE can't handle the module anymore when run directly - always run from the console.
-Clean up and bugfixes

PPHP 0.1.2

Changes:

  • __dom__ is no longer a keyword and lxml is no longer a dependency - it caused trouble when scripts were inside element parameters.
  • __script__ is a new keyword that contains the text of the current running script.
  • Fixed some bugs :)

PPHP Released

Python PPHP: Hypertext Preprocessor is a spinoff of PHP, in Python - use it to execute Python scripts inside HTML, much like PHP does.

To install it, open up a command processor (cmd on Windows or Terminal on *NIX or Mac) and type:

pip install pphp

See the installation instructions for more details.
